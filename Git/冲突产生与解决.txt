一、冲突产生原因
模拟：
a.同事小A在我下班后修改了readme文件
b.我第二天上班没有执行git pull，同时在本地仓库对readme文件做了修改，此时本地仓库和远程仓库的readme文件内容不一致，我修改完之后执行git push命令时提示产生了冲突（conflict）

二、解决
1、先执行git pull
注意：若此时又提示无法拉取代码，拉取前请先保存本地修改（也就是本地有其他未提交的修改文件发生冲突），可限制性git stash（将缓存这些文件的，缓存的文件是没有提交的），再次执行git pull，拉取完代码将缓存的文件恢复，执行 git stash pop恢复并将缓存堆栈中的对应stash删除

2、手动解决冲突的文件

3、重新提交冲突的文件
git commit -m "commit_info"

4、提交到远程仓库
git push

三、注意
执行git merge合并分支时容易产生冲突，解决方法也是一样
